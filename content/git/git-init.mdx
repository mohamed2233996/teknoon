## تجهيز مجلد خاص للمشروع

أي مشروع تنوي العمل عليه، قم بإنشاء مجلد خاص به و من ثم قم بوضع أي ملفات تابعة له بداخله.  
بعدها، إجعل Git تنشئ مستودع ( Repository ) خاص لمجلد المشروع.

هذا الأسلوب في العمل يجعلك قادر على تنظيم و مراقبة كل تغيير يحصل في المشروع.

  

### إنشاء مجلد للمشروع

قم بإنشاء مجلد فارغ إسمه `demo` لأننا سنقوم بوضع كل ملفات المشروع فيه لاحقاً.  
فيما يلي طريقة فعل ذلك بواسطة موجّه الأوامر أو الطرفية.

#### Terminal

\> mkdir demo  
\> cd demo

*   `mkdir` يستعمل لإنشاء المجلد.
*   `cd` يستعمل للإنتقال إليه.

## إنشاء مستودع خاص للمشروع

لتبدأ بتتبع ملفات المشروع عليك أولاً تنفيذ الأمر `git init` و أنت تقف بداخل مجلد المشروع الرئيسي كما يلي.

#### Terminal

\> git init  
Initialized empty Git repository in /home/workspaces/demo/.git/

بمجرد تنفيذ الأمر `git init` فإنه يتم إنشاء مجلد مخفي بداخل المشروع إسمه `.git` و هو يمثّل المستودع ( Repository ) الذي سيتم فيه حفظ كل التغييرات و الإصدارات الخاصة بالمشروع بشكل منظم جداً.

المجلد `.git` لا يمكنك رؤيته كباقي المجلدات إذا كانت إعدادات حاسوبك الإفتراضية هي عدم إظهار المجلدات المخفية.

## عرض المستودع المخفي في الطرفية

في حال كنت تستعمل الطرفية أو موجّه الأوامر و أردت رؤية المجلد `.git` الموجود في مشروعك فإنه عليك استعمال الأمر الذي يعرض الملفات و المجلدات المخفية. و بالطبع الأمر الذي عليك كتابته يختلف على حسب نظام التشغيل الذي تستعمله.

في حال كنت تستعمل نظام Windows أكتب `dir /a` و سيظهر لك جميع الملفات و المجلدات سواء كانت ظاهرة أو مخفية.

#### CMD

\> dir /a  
.git

في حال كنت تستعمل نظام Linux أو macOS أكتب `ls -a` و سيظهر لك جميع الملفات و المجلدات سواء كانت ظاهرة أو مخفية.

#### Terminal

\> ls -a  
.git
